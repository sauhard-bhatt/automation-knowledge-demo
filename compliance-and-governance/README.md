# Compliance and Governance

Temporary home for admin-only process assets.

This directory is kept in the same repository for demos. Later, it can be moved to a private admin repository.

## Files

- [00-operating-model.md](00-operating-model.md): How user-facing knowledge and admin governance work together.
- [01-standards.md](01-standards.md): Authoring and structure standards for incidents, doc-gaps, and runbooks.
- [02-review-checklist.md](02-review-checklist.md): Required checks before approving changes.
- [03-ownership-matrix.md](03-ownership-matrix.md): Roles and responsibilities for content lifecycle.
- [04-copilot-maintainer-instructions.md](04-copilot-maintainer-instructions.md): Guardrails for maintainers when using Copilot.
- [05-release-gate.md](05-release-gate.md): Criteria for merge and release readiness.
- [06-audit-log-template.md](06-audit-log-template.md): Lightweight template for recording governance decisions.
