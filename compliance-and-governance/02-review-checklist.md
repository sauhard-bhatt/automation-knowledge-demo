# Review Checklist

Use this checklist before approving any content change.

## Structural Checks

- [ ] File is in the correct canonical folder.
- [ ] File name follows id-based naming convention.
- [ ] Metadata keys are complete and valid.
- [ ] Related links resolve correctly.

## Content Checks

- [ ] Error signature is specific and testable.
- [ ] Fix steps are actionable and platform-accurate.
- [ ] Verify steps prove the fix actually worked.
- [ ] Doc-gap ownership is explicit.
- [ ] Migration runbooks include rollback.

## Index Checks

- [ ] Correct index file was updated.
- [ ] New artifact is discoverable from entry points.

## Safety Checks

- [ ] No secrets, tokens, or private endpoints committed.
- [ ] Commands are safe for intended environment.
- [ ] High-risk actions include warning notes when needed.
